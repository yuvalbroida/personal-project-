Contextual Supply Chain Scanner (Prototype)

## 📖 Product Overview
Legacy Application Security (AppSec) tools operate entirely blind to the deployment environment, resulting in massive alert fatigue that causes MLOps engineers to bypass security protocols. 

**The Echo Context Scanner** solves this friction. It is a headless CI/CD engine that cross-references application-layer vulnerabilities (e.g., PyPI packages) against the underlying Echo Base Image OS manifest. 

**The Value Proposition:** If a vulnerability requires an OS-level binary (like `bash` or `curl`) that cryptographically *does not exist* in Echo's minimal container, the engine dynamically downgrades the severity. This unblocks the developer's pipeline while generating a strict, unmodified JSON audit trail for the CISO.

## 🏗️ Architecture & Repository Contents

This repository serves as the functional prototype for Echo's Q1-26 library security wedge.

1. **`.github/workflows/echo-ai-scan.yml`**
   * **Purpose:** Demonstrates the frictionless, bottom-up adoption motion. Shows how an MLOps team integrates the tool in < 5 minutes.
2. **`echo_mock_engine.py`**
   * **Purpose:** The core Risk Engine API logic. Simulates the deterministic cross-referencing and dynamic severity downgrade mechanism. Includes a "fail-open" safety mechanism to protect build velocity.
3. **`echo_pipeline_output.json`**
   * **Purpose:** The machine-readable payload generated by the engine. Preserves the original CVSS scores to satisfy enterprise compliance audits.
